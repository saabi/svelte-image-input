# Internationalization (i18n)

Customize text content in svelte-image-input components to support multiple languages and localization.

## Overview

All user-facing text in the components can be customized through props, allowing you to:
- Support multiple languages
- Integrate with i18n libraries
- Customize text for specific use cases
- Improve accessibility with custom screen reader text

## ImageLoader Text Props

The `ImageLoader` component accepts the following text customization props:

- `prefixText`: Text displayed before the button (default: `"Drop, paste, or"`)
- `buttonText`: Text displayed on the load button (default: `"load an image"`)

## Usage

### Default (English)

If no text props are provided, the component uses English defaults:

```svelte
<ImageLoader onImageLoaded={handleImageLoaded} />
<!-- Displays: "Drop, paste, or load an image" -->
```

### Custom Text

Override text with custom strings:

```svelte
<ImageLoader
  onImageLoaded={handleImageLoaded}
  prefixText="Arrastra, pega o"
  buttonText="carga una imagen"
/>
<!-- Displays: "Arrastra, pega o carga una imagen" -->
```

### With ImageInput

The `ImageInput` component passes these props to its internal `ImageLoader`:

```svelte
<ImageInput
  bind:url
  prefixText="Drop, paste, or"
  buttonText="load an image"
/>
```

## Integration with i18n Libraries

### svelte-i18n

```svelte
<script>
  import { _ } from 'svelte-i18n';
  import { ImageInput } from 'svelte-image-input';
  
  let url = '';
  
  $: prefixText = $_('imageLoader.prefix');
  $: buttonText = $_('imageLoader.button');
</script>

<ImageInput
  bind:url
  {prefixText}
  {buttonText}
/>
```

Translation file (`en.json`):
```json
{
  "imageLoader": {
    "prefix": "Drop, paste, or",
    "button": "load an image"
  }
}
```

### i18next / svelte-i18next

```svelte
<script>
  import { useTranslation } from 'svelte-i18next';
  import { ImageInput } from 'svelte-image-input';
  
  const { t } = useTranslation();
  let url = '';
</script>

<ImageInput
  bind:url
  prefixText={t('imageLoader.prefix')}
  buttonText={t('imageLoader.button')}
/>
```

Translation file (`en.json`):
```json
{
  "imageLoader": {
    "prefix": "Drop, paste, or",
    "button": "load an image"
  }
}
```

### Custom i18n Solution

```svelte
<script>
  import { ImageInput } from 'svelte-image-input';
  import { getTranslation } from './i18n';
  
  let url = '';
  let locale = $state('en');
  
  $: prefixText = getTranslation(locale, 'imageLoader.prefix');
  $: buttonText = getTranslation(locale, 'imageLoader.button');
</script>

<ImageInput
  bind:url
  {prefixText}
  {buttonText}
/>
```

## Dynamic Language Switching

You can switch languages dynamically by updating the text props:

```svelte
<script>
  import { ImageInput } from 'svelte-image-input';
  
  let url = '';
  let locale = $state('en');
  
  const translations = {
    en: { prefix: 'Drop, paste, or', button: 'load an image' },
    es: { prefix: 'Arrastra, pega o', button: 'carga una imagen' },
    fr: { prefix: 'Glissez, collez ou', button: 'chargez une image' }
  };
  
  $: currentText = translations[locale];
</script>

<select bind:value={locale}>
  <option value="en">English</option>
  <option value="es">Español</option>
  <option value="fr">Français</option>
</select>

<ImageInput
  bind:url
  prefixText={currentText.prefix}
  buttonText={currentText.button}
/>
```

## Accessibility

When customizing text:

1. **Descriptive Text:** Ensure text clearly describes the action
2. **Screen Readers:** Text should be meaningful when read aloud
3. **Language Attributes:** Set the `lang` attribute on parent elements for proper screen reader pronunciation

```svelte
<div lang="es">
  <ImageLoader
    onImageLoaded={handleImageLoaded}
    prefixText="Arrastra, pega o"
    buttonText="carga una imagen"
  />
</div>
```

## Best Practices

1. **Consistent Terminology:** Use consistent terms across your application
2. **Context-Aware:** Consider the context where the component is used
3. **Length Considerations:** Be aware that some languages may have longer text
4. **RTL Support:** For right-to-left languages, you may need additional CSS adjustments

## Tips

- All text props are optional - components work with defaults
- You can override just one prop if needed
- Empty strings are handled gracefully
- Combine with theming for a fully localized experience
